*   **Backend Initialization:** Add a startup script in your backend (FastAPI/Flask) that runs a `CREATE TABLE IF NOT EXISTS` query to ensure the database is ready when the server boots.
*   **Storage Logic:** Update your chat endpoint so that every time a message is received from the user or generated by the AI, it is immediately executed as an `INSERT` statement into the SQLite table.
*   **History Retrieval:** Create a new GET endpoint (e.g., `/get_history`) that queries the database using `SELECT * FROM messages ORDER BY timestamp ASC` and returns the rows as JSON.

*   **File Icon Fix:** Create a hidden HTML `<input type="file">` element. Use a JavaScript `click()` event listener on your existing file icon to trigger the hidden input. Add an `onchange` listener to the input to handle the file upload via a `FormData` POST request to your backend.
*   **Finance Mode Toggle:** Add a visual switch or button (e.g., "Switch to Finance Mode"). 
*   **State Management:** Create a JavaScript variable `isFinanceActive` (defaulting to false). When the button is toggled, update this variable and change the UI theme (e.g., green borders or a badge) to indicate the mode is active.
*   **Session Hydration:** Add an `window.onload` function that calls the `/get_history` endpoint. Iterate through the returned data to render all previous chat bubbles automatically when the user opens or refreshes the page.

### **3. Backend Logic & Mode Routing**
*   **Request Payload Update:** Modify the frontend "Send" function to include the current mode in the JSON body: `{"message": "...", "mode": "finance"}`.
